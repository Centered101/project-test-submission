เอกสารของ การเก็บข้อมูลแบบ-Local-Storage

อัพเดทข้อมูล 2025-08-28

ภาพรวม

Local Storage คือพื้นที่เก็บข้อมูลในเบราว์เซอร์ที่ช่วยให้เว็บแอปพลิเคชันสามารถจัดเก็บข้อมูลผู้ใช้ได้ในรูปแบบ Key-Value โดยข้อมูลที่เก็บจะไม่ถูกลบแม้ผู้ใช้จะปิดเบราว์เซอร์หรือรีสตาร์ทเครื่อง เหมาะสำหรับการเก็บข้อมูลชั่วคราว เช่น การตั้งค่าผู้ใช้ หรือสถานะบางอย่างของแอปพลิเคชัน

คุณสมบัติ

  • เก็บข้อมูลแบบ Key-Value: ข้อมูลถูกจัดเก็บในคู่ key และ value
  • ไม่มีวันหมดอายุ: ข้อมูลจะอยู่จนกว่าผู้ใช้หรือโปรแกรมจะลบออก
  • พื้นที่จัดเก็บ: จำกัดพื้นที่ประมาณ 5MB ต่อโดเมน
  • เหมาะสำหรับข้อมูลไม่สำคัญ: เช่น การตั้งค่าธีมหรือสถานะผู้ใช้
  • ไม่ปลอดภัยสำหรับข้อมูลสำคัญ: เพราะสามารถเข้าถึงได้ง่ายผ่านเครื่องมือเบราว์เซอร์

ข้อดี

  • ใช้งานง่ายด้วย JavaScript
  • ทำงานได้เร็ว เพราะไม่ต้องเรียกใช้เซิร์ฟเวอร์​
  • เหมาะสำหรับเก็บข้อมูลที่ไม่สำคัญ เช่น การตั้งค่าผู้ใช้ หรือสถานะชั่วคราว

ข้อเสีย

  • เก็บข้อมูลได้จำกัด
  • ข้อมูลไม่ปลอดภัยสำหรับการใช้งานสำคัญ
  • ต้องการโดเมนเดียวกัน (Same-origin policy)

Local Storage เหมาะสำหรับเว็บที่ต้องการจัดเก็บข้อมูลผู้ใช้แบบพื้นฐานและต้องการลดการพึ่งพาเซิร์ฟเวอร์! 😊

Local Storage คือพื้นที่เก็บข้อมูลของเว็บเบราว์เซอร์ที่ช่วยให้เว็บแอปพลิเคชันสามารถจัดเก็บข้อมูลได้บนอุปกรณ์ของผู้ใช้แบบถาวร โดยไม่ต้องใช้ฐานข้อมูลหรือเซิร์ฟเวอร์ในบางกรณี ข้อมูลที่เก็บไว้ใน Local Storage จะยังคงอยู่แม้ปิดเบราว์เซอร์หรือรีสตาร์ทเครื่อง (ยกเว้นผู้ใช้ลบข้อมูลด้วยตนเอง)
